  • Turn messy numbers into clean pictures + short paragraphs.
  • Categorical displays (bar/stacked bar, contingency tables).
  • Quantitative displays (histograms, boxplots, timeplots).
  • Shape–center–spread; Normal model; correlation & regression.
  • Ch 1–2: data + categorical stories.
  • Ch 3–5: quantitative stories & z-scores.
  • Ch 6–8: tying together and adding relationships (scatterplots, regression).

Ch 15–18One-Sample Inference
CI and tests for one proportion or one mean▾
Ch 15Confidence Intervals for Proportions
p̂ ± z*SE(p̂)
  • Estimate a population proportion p using a confidence interval from a sample.
  • CI: p̂ ± z*·SE(p̂)
  • SE(p̂) = √[p̂(1−p̂)/n]
  • Confidence level determines z* (e.g., 95% → 1.96).
p̂ = x/n
  • Conditions: random/independent sample, success/failure (np̂, n(1−p̂) ≥ about 10).
  • Margin of error = z*·SE.
  • State p clearly (“true proportion of …”).
  • Interpret CI as a range of plausible values for p, not as a probability statement about p.
  • Exam: compute CI, interpret it, and compare widths when n or confidence level changes.
Ch 16Testing Hypotheses About Proportions
One-sample z test
  • Test a claim about p using P-values rather than just a point estimate.
  • H₀: p = p₀ vs Hₐ: p > p₀, p < p₀, or p ≠ p₀.
  • z = (p̂ − p₀) / SE₀ with SE₀ = √[p₀(1−p₀)/n].
  • P-value from standard Normal.
  • Conditions: same as CI but using p₀ in SE and success/failure.
  • Connection: 2-sided test ↔ whether p₀ is inside CI.
  • Four steps: State (H₀, Hₐ), Plan (conditions), Do (z, P), Conclude (in context).
  • Explain P-value as “probability of a result this extreme if H₀ were true”.
  • Exam: run full test; interpret both statistical and practical significance.
Ch 17More About Tests
Type I/II errors & power
  • Recognize that hypothesis tests can be wrong and quantify that risk.
  • Type I error: rejecting true H₀ (probability α).
  • Type II error: failing to reject a false H₀.
  • Power = 1 − β = chance to detect a real effect.
  • Power increases with n, larger α, larger true effect, smaller variability.
  • One-sided vs two-sided tests trade-off power vs flexibility.
  • Describe Type I/II in the real scenario (“We wrongly conclude …”).
  • Reason qualitatively about how to increase power.
  • Exam: label error types correctly and suggest design changes that increase power.
Ch 18Inferences About Means
t-models for μ
  • Estimate or test a population mean μ when σ is unknown.
  • t-distribution with df = n−1.
  • CI: x̄ ± t*·SE(x̄), SE(x̄) = s/√n.
  • Test: t = (x̄ − μ₀)/SE(x̄).
Use t instead of z when σ is unknown.
  • Conditions: independence and Nearly Normal population or large n.
  • Use plots to justify Normality, not just “n ≥ 30”.
  • Follow the same four-step structure; always include units.
  • Compare t-based result conceptually to z-approximation.
  • Exam: compute and interpret 1-sample t CI/test; comment on whether conditions are reasonable.
Ch 19–23Comparisons & Associations
Two samples, counts, regression inference▾
Ch 19Comparing Means
Two-sample t
  • Compare average outcomes of two independent groups (μ₁ vs μ₂).
  • Statistic: x̄₁ − x̄₂; parameter: μ₁ − μ₂.
  • SE(x̄₁ − x̄₂) = √(s₁²/n₁ + s₂²/n₂).
  • Two-sample t CI/test.
  • Conditions: independence within and between groups; Nearly Normal within each group or large n.
  • Check design first: are groups independent or paired?
  • Exam: interpret CI/test for μ₁−μ₂ including sign, size, and units.
Ch 20Paired Samples and Blocks
Matched pairs
  • Use differences when measurements are naturally paired or blocked.
  • Create d = (after − before) or similar.
  • Use one-sample t on d̄.
  • Designs: before/after, matched pairs, blocking to reduce variation.
  • Plot differences, not raw scores, to check Nearly Normal for d.
  • Exam: correctly identify paired design and set up d-based analysis.
Ch 21Comparing Two Proportions
Two-proportion z
  • Compare two population proportions p₁ and p₂.
  • Statistic: p̂₁ − p̂₂; parameter: p₁ − p₂.
  • Test SE uses pooled p̂ when H₀: p₁ = p₂.
p̂_pool = (x₁+x₂)/(n₁+n₂)
  • Conditions: independence & success/failure in each group.
  • Use unpooled SE for CI; pooled SE only for the test under H₀.
  • Exam: carry out test/CI and interpret the difference p₁−p₂ in plain language.
Ch 22Comparing Counts
Chi-square
  • Use χ² tests to compare observed counts to expectations from a model.
  • χ² = Σ (O−E)²/E.
  • Goodness-of-fit, independence, and homogeneity versions.
  • Compute expected counts from model or margins.
  • Interpret large cell contributions to χ² as “where the model fails”.
  • Check expected counts ≥ about 5.
  • Use standardized residuals to see which cells drive the result.
  • Exam: choose right χ² test; interpret conclusion and key residuals in context.
Ch 23Inferences for Regression
Slope tests & CI
  • Decide whether there is a real linear relationship between x and y in the population.
  • Parameter: β₁ (true slope), estimate: b₁.
  • Test H₀: β₁ = 0 using t = b₁/SE(b₁).
  • CI: b₁ ± t*·SE(b₁).
  • Conditions: linearity, independent errors, Normal residuals, equal spread.
  • Check residual plot and histogram/Normal plot of residuals.
  • Always read the residual plot before trusting tests/CI for slope.
  • Interpret slope and CI in context (change in y per 1 unit of x).
  • Exam: read regression output; test whether there is evidence of association; explain in words.
Part VI – Assessing Associations Between Variables (Ch 24–27)
ANOVA and multiple regression as one “general linear model” family.
Part Tree
  • Move beyond simple two-group or simple linear comparisons to richer association models.
  • ANOVA and regression share the form: response = model + random error.
  • One-way ANOVA; multifactor ANOVA with interactions.
  • Multiple regression with several predictors; diagnostics and model-building.
  • Decomposing variation (SS, MS, F); partial slopes; collinearity; leverage and influence.
  • Always check residuals and assumptions, not just p-values and R².
  • Exam: interpret ANOVA and regression tables; explain what they say about associations.
Ch 24–27Chapter Trees
ANOVA & multiple regression▾
Ch 24Analysis of Variance
Comparing many means
  • Test whether several group means are all equal.
  • Decompose variation: between vs within groups.
  • F = MS_between / MS_within.
  • One-way layout; equal-variance assumption; Normal within groups.
  • Read ANOVA table; if significant, consider follow-up comparisons.
  • Exam: interpret F, df, and P-value and say what “at least one mean differs” means.
Ch 25Multifactor ANOVA
Interactions
  • Study how two or more factors and their interaction affect a response.
  • Main effects for each factor; interaction term for combined effect.
  • Interaction: effect of one factor depends on the level of another.
  • Interaction plots show non-parallel lines when interaction is present.
  • When interaction is strong, interpret effects at specific levels of the other factor.
  • Exam: recognize and describe interactions from graphs or tables.
Ch 26Multiple Regression
Many predictors
  • Model a response using several predictors at once.
  • Each slope is a partial effect, controlling for other predictors.
  • Adjusted R² for model comparison.
  • Collinearity; redundant predictors; indicator (dummy) variables.
  • Interpret coefficients carefully when predictors are correlated.
  • Use residual plots to check model form and assumptions.
  • Exam: interpret regression output with multiple predictors; decide if a predictor is significant.
Ch 27Multiple Regression Wisdom
Diagnostics & model building
  • Good regression practice = skepticism + diagnostics, not just pretty output.
  • Residual analysis; leverage vs influence; overfitting; variable selection.
  • Outliers vs high leverage vs influential observations.
  • Comparing models with and without certain predictors.
  • Never rely on R² alone; always inspect residuals and influential cases.
  • Exam: explain why a model is or is not appropriate and suggest basic improvements.
Part VII – Modelling the World at Large (Ch 28–29)
Nonparametric tests and bootstrap when parametric assumptions fail.
Part Tree
  • Provide flexible tools for messy, non-Normal data.
  • Trade a bit of power for safer assumptions.
  • Rank-based nonparametric tests.
  • Bootstrap resampling for confidence intervals.
  • When classic t/ANOVA assumptions break badly, use these robust options.
  • Check for severe skew/outliers and small to moderate n before deciding.
  • Exam: pick robust methods and justify why they’re safer than Normal-based tools.

Ch 28Nonparametric Tests
Rank-based & sign tests
  • Provide inference tools that don’t rely heavily on Normality.
  • Use ranks or signs instead of raw values.
  • Less sensitive to extreme outliers or heavy skew.
  • Examples: sign test, Wilcoxon tests (depending on course emphasis).
  • Decide to switch when distribution is very skewed or heavily outlier-contaminated.
  • Exam: explain when and why you’d choose a rank-based method instead of a t-test/ANOVA.
Ch 29The Bootstrap
Resampling for CIs
  • Use the sample as a stand-in for the population by resampling with replacement.
  • Build a bootstrap distribution of the statistic (mean, median, difference, etc.).
  • Take percentiles (or similar) of that distribution to form a CI.
  • Algorithm: resample many times, compute statistic each time, then summarize.
  • Sketch bootstrap distributions; note skew and spread; explain what the CI means.
  • Exam: outline bootstrap steps from scratch; interpret the resulting CI and compare to t-based CI.
