Stats Starts Here – Chapter 1 Tutorial

1.1 What Is Statistics?

Key ideas

  • Statistics is a way of reasoning plus tools for learning from data amid variation and uncertainty.
  • Data are raw values that only become meaningful when tied to context.
  • Variation is inevitable; statistical thinking looks past noise to reveal patterns.
  • The W’s (Who, What, When, Where, Why, How) anchor any data story.

Important formulas & their meaning

  • Statistics (discipline): Reasoning + methods that turn data into understanding; moves from questions to evidence-based conclusions.
  • Data: Recorded values with context (W’s); context transforms numbers or labels into information about reality.
  • Statistic (calculation): Numerical summary (e.g., mean, proportion) derived from data to answer a focused question.

Short example(s)

  • Texting and driving: Simulator experiment measured reaction times of sober, drunk, and texting drivers (Who: drivers; What: reaction time; Why: safety). Finding: texting drivers reacted slowest.
  • Targeted ads: Platforms use profile data (age, interests, friends) to predict purchases, showing how context plus data drives decisions.

1.2 Data

Key ideas

  • Context via the W’s makes data interpretable.
  • Data tables organize information: rows = cases (who), columns = variables (what).
  • Samples represent populations; representativeness is crucial for reliable inference.
  • Collection method (How) can make results insightful or meaningless.

Important formulas & their meaning

  • Case/individual: The entity each row describes; keeps all measurements for one unit together.
  • Variable: A recorded characteristic of cases; columns hold traits we compare or summarize.
  • Population vs. Sample: Population is the full group of interest; a sample is the observed subset used to infer about that group.
  • Record: Digital row that preserves all values for one case.

Short example(s)

  • Amazon orders: Unlabeled values become meaningful in a table with columns like Order Number (ID), Name, Province, Price ($), Area Code, Gift?, ASIN, Artist. Now we can study pricing or shipping choices.

1.3 Variables

Key ideas

  • Variable type (categorical vs. quantitative) determines suitable summaries and graphs.
  • Units give quantitative variables meaning; numeric labels aren’t always quantitative.
  • Identifier variables label cases uniquely; ordinal variables show order without fixed spacing.
  • Always check the study’s Why before assigning type, especially when numbers label categories.

Important formulas & their meaning

  • Categorical variable: Groups cases (e.g., province, shipping method); best summarized by counts or proportions.
  • Quantitative variable: Numerical measure with units (e.g., price in dollars, reaction time in seconds); supports arithmetic like averages.
  • Identifier variable: Unique label per case (e.g., transaction number, student ID); used for tracking or joining tables, not analysis.
  • Ordinal variable: Ordered categories without equal intervals (e.g., “poor” < “fair” < “good” < “excellent”); order matters, arithmetic is risky.
  • Measurement unit: The standard step size (dollars, hours, km/h) that makes quantitative values interpretable.

Short example(s)

  • Shipping method: Ground vs. 2-day vs. Overnight is categorical; we count orders in each group.
  • Tablet review: Manufacturer (categorical), price ($, quantitative), battery life (hours, quantitative), OS (categorical), performance score (quantitative index), memory card reader (categorical). Variable types guide which comparisons make sense.
  • Course rating 1–5: Ordered satisfaction levels; higher numbers mean better, but gaps aren’t equal—treat as ordinal.
